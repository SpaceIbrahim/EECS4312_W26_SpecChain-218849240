# Requirement ID: FR1

- Description: [The system shall support uninterrupted bedtime playback so that users can listen to sleep content without the session stopping unexpectedly.]
- Source Persona: [Mubarak a tired father]
- Traceability: [Derived from review group G5]
- Acceptance Criteria: [Given the user starts sleep audio and leaves the device idle, when playback is in progress, then the session shall continue until the content ends or the user stops it.]

# Requirement ID: FR2

- Description: [The system shall provide short guided sessions for stress relief that can be completed in ten minutes or less.]
- Source Persona: [Noor, an Overwhelmed Professional]
- Traceability: [Derived from review group G1]
- Acceptance Criteria: [Given the user wants a short stress-relief session, when they browse available guided sessions, then the system shall provide sessions whose duration is ten minutes or less.]

# Requirement ID: FR3

- Description: [The system shall clearly display whether content is free, part of a trial, or requires a paid subscription before the user starts it.]
- Source Persona: [Ibrahim a Cautious Subscriber]
- Traceability: [Derived from review group G2]
- Acceptance Criteria: [Given the user views a piece of content, when its details are shown, then the system shall indicate whether the content is free, included in a trial, or requires a paid subscription before playback starts.]

# Requirement ID: FR4

- Description: [The system shall present subscription terms, billing frequency, trial duration, and renewal conditions before the user confirms a subscription or free trial.]
- Source Persona: [Ibrahim a Cautious Subscriber]
- Traceability: [Derived from review group G2]
- Acceptance Criteria: [Given the user is about to activate a trial or subscription, when the confirmation screen is displayed, then the system shall show the price, billing frequency, trial duration, and renewal terms before the user confirms.]

# Requirement ID: FR5

- Description: [The system shall allow users to cancel a trial or subscription through a clearly accessible cancellation path.]
- Source Persona: [Ibrahim a Cautious Subscriber]
- Traceability: [Derived from review group G2]
- Acceptance Criteria: [Given the user has an active trial or subscription, when they open subscription management, then the system shall provide a visible option to cancel it.]

# Requirement ID: FR6

- Description: [The system shall allow downloaded audio content to remain accessible when the device has no internet connection.]
- Source Persona: [Zain an Offline Listener]
- Traceability: [Derived from review group G3]
- Acceptance Criteria: [Given the user has downloaded audio and the device is offline, when they open the downloaded content and press play, then the system shall play the audio without requiring an internet connection.]

# Requirement ID: FR7

- Description: [The system shall play downloaded content offline without skipping, restarting unexpectedly, or showing a connection error.]
- Source Persona: [Zain an Offline Listener]
- Traceability: [Derived from review group G3]
- Acceptance Criteria: [Given the user plays previously downloaded audio while offline, when playback begins, then the session shall continue without skipping, restarting, or displaying a connection error.]

# Requirement ID: FR8

- Description: [The system shall provide search and category navigation that allows users to find meditation, sleep, and sound content efficiently.]
- Source Persona: [Sobia a frustrated user]
- Traceability: [Derived from review group G4]
- Acceptance Criteria: [Given the user searches for a meditation, sleep story, or soundscape, when they use search or browse categories, then the system shall return relevant results for that content type.]

# Requirement ID: FR9

- Description: [The system shall allow users to access previously completed, in-progress, downloaded, or favourited content from dedicated sections.]
- Source Persona: [Sobia a frustrated user]
- Traceability: [Derived from review group G4]
- Acceptance Criteria: [Given the user wants to return to content they used before, when they open the library or profile area, then the system shall provide separate sections for completed, in-progress, downloaded, or favourited content.]

# Requirement ID: FR10

- Source Persona: [Mubarak a tired father]
- Traceability: [Derived from review group G5]
